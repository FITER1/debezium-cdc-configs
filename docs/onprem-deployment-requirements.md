# CDC On-Premise Deployment — Outstanding Requirements

The following items must be resolved before the Debezium Oracle XStream CDC workload can be deployed to Access Bank's on-premise Kubernetes cluster. The CDC service captures change events from Oracle Flexcube (on-prem) and streams them to AWS MSK (Kafka) over VPN, where Synapse consumes them on EKS. The deployment uses the Strimzi Operator to manage Kafka Connect via `KafkaConnect` and `KafkaConnector` custom resources.

| # | Requirement | Purpose | How to Obtain |
|---|-------------|---------|---------------|
| 1 | **MSK Bootstrap Servers** | Kafka broker address that the on-prem Kafka Connect uses to produce CDC events over VPN. Configured in the `KafkaConnect` CR `bootstrapServers` field and each `KafkaConnector` CR's `schema.history.internal` config. | AWS Console → Amazon MSK → Cluster → "View client information" → copy the "Private endpoint" under TLS. Format: `b-1.xxx.kafka.eu-west-1.amazonaws.com:9094,b-2.xxx...` |
| 2 | **Oracle CDB Name** | Debezium XStream requires the Container Database name to establish the OCI capture connection. Maps to `database.dbname` in each `KafkaConnector` CR config. | DBA runs: `SELECT NAME FROM V$DATABASE;` while connected to the CDB root (not the PDB). The returned value is what we need. |
| 3 | **MSK mTLS Certificates** | Kafka Connect authenticates to MSK using mutual TLS. The JKS files are stored in a K8s Secret (`cdc-msk-certs`) and mounted via Strimzi's `externalConfiguration` at `/opt/kafka/external-configuration/msk-certs/`. | Generate a Private CA in AWS Certificate Manager (ACM PCA), issue a client certificate, then package into JKS files: `keytool -importkeystore -srckeystore client.p12 -destkeystore keystore.jks` and `keytool -import -alias msk-ca -file msk-ca.pem -keystore truststore.jks`. Record the 3 passwords. Create the K8s Secret: `kubectl create secret generic cdc-msk-certs -n cdc --from-file=truststore.jks --from-file=keystore.jks` |
| 4 | **AWS Secrets Manager — Create Secrets** | The ExternalSecret controller syncs these into K8s Secret `cdc-extsec` that Strimzi mounts via `externalConfiguration` volumes for runtime credential access. Without them, the pod will fail to start. | An IAM user/role with `secretsmanager:CreateSecret` runs: `aws secretsmanager create-secret --name "cdc/oracle-credentials" --secret-string '{"password":"..."}' --region eu-west-1` and `aws secretsmanager create-secret --name "cdc/msk-credentials" --secret-string '{"truststore_password":"...","keystore_password":"...","key_password":"..."}' --region eu-west-1` |
| 5 | **VPN Network Path (On-prem → MSK)** | CDC events flow from on-prem pods to MSK over the site-to-site VPN. If the network path is broken, connectors will fail with `BrokerNotAvailableException`. | 1. Confirm VPN tunnel is UP (AWS VPN Console → Site-to-Site → Status). 2. Add on-prem pod CIDR to MSK security group inbound rule (port 9094). 3. Test from a pod: `openssl s_client -connect <msk-broker>:9094` |
| 6 | **On-prem Kubernetes Cluster Access** | Required to deploy the Strimzi Operator, CDC namespace, `KafkaConnect` CR, `KafkaConnector` CRs, and ExternalSecret to the on-prem cluster. | Provide one of: (a) kubeconfig file for the on-prem cluster, (b) ArgoCD Application pointing to the `workloads/cdc/overlays/dev/` path, or (c) SSH/VPN access to a bastion with `kubectl` configured. |
| 7 | **IAM User for On-prem AWS Access** | A single IAM user in account `228736141087` serves two purposes on-prem: (1) pulls the CDC image from ECR (one-time per version), and (2) allows the ExternalSecrets controller to read CDC secrets from AWS Secrets Manager. One credential to manage for all AWS access from on-prem. | Create IAM user `cdc-onprem-service` with policy: `ecr:GetAuthorizationToken`, `ecr:BatchGetImage`, `ecr:GetDownloadUrlForLayer` (Resource: `*`) and `secretsmanager:GetSecretValue`, `secretsmanager:DescribeSecret` (Resource: `arn:aws:secretsmanager:eu-west-1:228736141087:secret:cdc/*`). Generate access keys. Store on-prem as: (a) K8s `imagePullSecret` for ECR, (b) K8s Secret for the `ClusterSecretStore` credential. |
| 8 | **Strimzi Operator on On-prem K8s** | The Strimzi Cluster Operator manages the Kafka Connect lifecycle via `KafkaConnect` and `KafkaConnector` CRDs. It creates the Deployment, handles connector registration, autoRestart, and status reporting. | Install via Helm: `helm install strimzi-cluster-operator oci://quay.io/strimzi-helm/strimzi-kafka-operator --namespace strimzi --create-namespace`. Verify: `kubectl get crd kafkaconnects.kafka.strimzi.io` |
| 9 | **Oracle Network Reachability from Pods** | XStream uses a persistent OCI connection to Oracle. If the pod cannot reach the DB, or if firewalls kill idle connections, the connector will crash and require a full re-snapshot. | From a test pod in the `cdc` namespace: `nc -zv AB02DWDB-SCAN3.accessbankplc.com 6655`. Ensure firewall rules allow long-lived TCP (set idle timeout ≥ 30 min or disable it). Verify SCAN listener resolves: `nslookup AB02DWDB-SCAN3.accessbankplc.com` |
